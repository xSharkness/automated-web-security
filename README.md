# 🛡️ Web Security Scanner

[![Python](https://img.shields.io/badge/Python-3.7%2B-blue.svg)](https://www.python.org/)
[![PyQt6](https://img.shields.io/badge/GUI-PyQt6-green.svg)](https://pypi.org/project/PyQt6/)
[![SQLite](https://img.shields.io/badge/DB-SQLite-orange.svg)](https://www.sqlite.org/)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

**Web Security Scanner** — десктопное приложение для автоматического тестирования безопасности веб-приложений, объединяющее три ключевых подхода: **SCA** (анализ зависимостей), **SAST** (статический анализ кода) и **DAST** (динамический анализ веб-логики). Система интегрируется с открытыми банками уязвимостей (NVD/CVE, MITRE CWE, OWASP Top 10) и формирует структурированные отчёты с рекомендациями по устранению найденных проблем.

---

## ✨ Возможности

- 🔍 **Комплексный анализ безопасности** — объединение SCA, SAST и DAST в едином процессе
- 📡 **Автоматическая синхронизация** с NVD, MITRE CWE и OWASP Top 10 через REST API
- 📊 **Количественная оценка риска** на основе метрики CVSS
- 🏷️ **Классификация уязвимостей** по CWE и OWASP Top 10
- 📁 **Рекурсивный обход проекта** с парсингом файлов манифестов (`requirements.txt`, `package.json`, `pom.xml`, `go.mod`, `composer.json` и др.)
- 🧪 **Динамическая верификация** — подтверждение уязвимостей отправкой HTTP-запросов с пейлоадами
- 📄 **Генерация HTML-отчёта** с рекомендациями по устранению
- 🖥️ **Удобный GUI** на PyQt6 с вкладками «История», «Уведомления» и экспортом отчётов
- 💾 **Локальная БД SQLite** — приложение полностью переносимо и не требует настройки сервера

📖 Использование
Запустите приложение:
bash
1
Дождитесь синхронизации с MITRE CWE и NVD (при первом запуске может занять несколько минут).
Укажите параметры целевого приложения:
Путь к каталогу с исходным кодом (для SAST)
IP-адрес и порт работающего веб-приложения (для DAST)
Нажмите «Сканировать» — система последовательно выполнит:
SCA — анализ зависимостей на известные CVE
SAST — поиск опасных паттернов в коде (SQLi, XSS, Command Injection и др.)
DAST — динамическая верификация найденных уязвимостей
Просмотрите результаты на вкладках «История» и «Уведомления».
Экспортируйте отчёт в HTML через кнопку «Экспорт отчёта».

🗄 Структура базы данных
Система использует локальную БД vulnerabilities.db со следующей схемой:
Таблица
Назначение
CVE
Хранилище уязвимостей (ID, описание, CVSS, severity)
CPE
Затронутые продукты, вендоры и версии
CWE
Классификатор типов уязвимостей
OWASP
Категории OWASP Top 10
CVE_CWE
Связь CVE ↔ CWE
CWE_OWASP
Связь CWE ↔ OWASP
SCAN_RESULTS
Результаты сканирования (SCA/SAST/DAST)
PRODUCT_ALIASES
Нормализация названий продуктов
SYNC_STATE
Состояние синхронизации с NVD (инкрементальное обновление)
---

📚 Источники данных
- NVD (National Vulnerability Database) — основной источник CVE с метриками CVSS и CPE-записями
- MITRE CWE — классификатор типов уязвимостей
- OWASP Top 10 — классификация бизнес-рисков веб-приложений
